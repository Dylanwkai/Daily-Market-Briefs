# Markets Second Brain — Workflow

Obsidian vault. Four folders:

- `briefings/` — daily PDFs from the scheduled Cowork task (drag in weekly, or point the task's output folder here)
- `journal/` — ONE file per month (`Journal_YYYY-MM.md`), appended daily. Never one file per day.
- `ideas/` — one file per thesis, from `templates/Idea.md`. Only ideas that survived a stress-test live here.
- `INDEX.md` — one line per idea. **The most important file in the vault** — it's what makes retrieval cheap.

---

## Daily loop (~20 min)

**1. Read** the briefing PDF (7:00 AM SGT).

**2. Journal** — open this month's journal file, add today's entry (template at bottom of the monthly file):
- 3-line take on the day
- Your answer to the Journal Seed question
- Sparks: anything from Idea Lab / Trade Floor / Radar worth developing

**3. Stress-test (only when you have a spark)** — open a **NEW Cowork chat** (never reuse yesterday's) and paste this opener:

> Attached: INDEX.md and today's journal entry. Stress-test this thesis — play devil's advocate, don't agree with me.
> **Idea:** [long/short] [ticker/instrument]
> **Thesis (3 lines):** ...
> **Why now:** ...
> **What I think is mispriced:** ...
> Attack in this order: (1) what's actually priced in, (2) who is on the other side and why they might be right, (3) catalyst timing, (4) risk/reward math, (5) what kills it. Then give a verdict: develop / park / kill.

Attach only: `INDEX.md` + today's journal entry (copy-paste the entry, not the whole month). Nothing else.

**4. If the idea survives** — in the same chat say: *"Survived. Fill out the idea template and give me the INDEX line."* Paste the output into a new file in `ideas/` and add the line to `INDEX.md`. Close the chat.

**5. If it dies** — one line in the journal entry: what killed it. Dead ideas are interview material too ("here's a thesis I killed and why").

---

## Weekly (Saturday, 15 min)
- Update `status:` in each open idea file (developing → open → closed/stopped) and log what moved
- Score closed ideas in the Status log: right/wrong on thesis AND right/wrong on price — separately. Being right for the wrong reason goes in the log.

## Pre-interview
New chat: *"Attached INDEX.md. Pick my 5 strongest ideas for a [buyside/S&T] interview, then grill me on each."* Attach the specific idea files only for the 5 it picks.

---

## Token rules
1. New chat per discussion; close it when done. Never a running mega-chat.
2. Attach only INDEX.md + the specific entry/idea file under discussion.
3. Everything in markdown. PDFs are for reading; markdown is for retrieval.
4. Journal = one monthly file, appended daily. Update the doc, don't multiply docs.
